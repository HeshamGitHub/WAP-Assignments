Sevlet Quiz Number 1

  1. What is the difference between a web server and a web container?
    Web Server : where the browsers or any http client can communicate, Http server implmentation Web container : is handling Web components life cycle , creating thread per each request
  2. What is a servlet?
    is a web component which run on server side , Servlet is a Java class that extends httpservlet , which has configuration in Web.xml
  3. How do web servers and web containers interact with servlets?
    Web Servers recieve Http requests from the Http clients like browsers , the communicate with the Web container asking for the execution of the target Servlet, container will create a new thread of the target sevlet and create Request Object and then pass it for the doService , after execution container send back the final output response for the web server which directly reply back with Http resonse formate to the caller,
  4. Who creates request objects?
    Web Container
  5. What are the states in the servlet lifecycle?
    Instation - Intiation - Service - Destory
  6. Who calls init and when?
    Web Container , on the first request of that Servlet
  7. Which of init, service, and doGet should you override?
    we can Override all, however we usually override DoGet
  8. In what sense are servlets multi-threaded?
    Container creates one thread for handling each request
  9. What are the implications of this for servlet instance variables?
    this is not thread safe as Servlet is handling requests by multithreading,
